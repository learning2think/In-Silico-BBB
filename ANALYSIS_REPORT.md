## 1. Резюме проекта

**In Silico BBB** — вычислительная платформа для моделирования транспорта малых молекул через
гематоэнцефалический барьер (ГЭБ). Ключевая идея: заменить/дополнить дорогостоящие in vitro
эксперименты (PAMPA-BBB, клеточные монослои) быстрым механистическим расчётом на основе
двухкомпартментной кинетической модели.

### Архитектура системы (текущее состояние)

### Математическая модель — краткое изложение

Двухкомпартментная система ОДУ с двумя транспортными механизмами:

| Механизм | Уравнение | Параметры |
|---|---|---|
| Пассивная диффузия (Фик) | `J_diff = k_pass × (C_blood − C_brain)` | k_pass [1/ч] |
| Активный эффлюкс P-gp (Михаэлис–Ментен) | `J_pgp = Vmax × C_brain / (Km + C_brain)` | Vmax [мкМ/ч], Km [мкМ] |

**Выходные метрики:** Cmax (мозг), AUC (мозг), Kp,uu (равновесное соотношение мозг/кровь).

## 2. Технический критический анализ

### 2.1 Сильные технические решения

#### Выбор численного метода — LSODA
**Оценка: корректно.** Использование LSODA через `scipy.integrate.solve_ivp` оправдано:
система ОДУ потенциально жёсткая при малых Km (< 1 мкМ) и высоких концентрациях — P-gp
работает в насыщенном режиме и создаёт сингулярность. LSODA автоматически переключается
Adams ↔ BDF без вмешательства пользователя. Точность (rtol=1e-8, atol=1e-10) избыточная
для практических задач, но гарантирует корректность.

#### Закон сохранения вещества
**Оценка: корректно.** Коэффициент V_brain/V_blood явно введён в правые части ОДУ,
что гарантирует: количество молекул, покидающих кровь, точно равно поступающим в мозг
(и наоборот для P-gp). Это критически важно для корректного расчёта Kp,uu.

#### Модульная архитектура
**Оценка: хорошо.** Разделение на `core/model.py` (физика) → `core/simulator.py` (решатель) →
`app.py` (UI) позволяет тестировать и расширять каждый слой независимо.

#### Защита от отрицательных концентраций
```python
c_brain_safe = max(c_brain, 0.0)  # в model.py:76
```
**Оценка: корректно.** Численные решатели могут дать незначительно отрицательные значения
на жёстких участках — защитный зажим предотвращает физически бессмысленный результат.

### 2.2 Критические ограничения модели

#### Ограничение 1: k_pass — агрегированный параметр
**Серьёзность: Высокая**

k_pass в модели — безразмерный коэффициент `[1/ч]`, агрегирующий три биофизических
параметра: **проницаемость мембраны × площадь поверхности / объём компартмента**:

```
k_pass (модель) ≈ P_app × S / V   [см/с × см² / мл → условные ед.]
```

Это означает, что параметр нельзя напрямую сопоставить с:
- Papp из PAMPA-BBB ассей (единицы: ×10⁻⁶ см/с)
- PS (перфузионный клиренс) из ПЭТ-исследований (единицы: мл/мин/г)

**Последствие:** сложность валидации на экспериментальных данных без пересчётного
коэффициента; пользователи могут некорректно интерпретировать числовые значения параметра.

**Рекомендация:** ввести явную формулу пересчёта Papp (PAMPA) → k_pass и задокументировать её.

---

#### Ограничение 2: Kp,uu рассчитывается без учёта связывания с белками
**Серьёзность: Высокая**

Текущий расчёт в `app.py:272–276`:
```python
kp_uu = (
    result.c_brain[-n_tail:].mean()
    / result.c_blood[-n_tail:].mean()
)
```

Формально корректный **истинный Kp,uu** вычисляется как:
```
Kp,uu = (C_brain_total × fu_brain) / (C_blood_total × fu_plasma)
```

Где:
- `fu_brain` — несвязанная фракция в ткани мозга (типично 0.01–0.3)
- `fu_plasma` — несвязанная фракция в плазме (типично 0.01–1.0)

Без этих параметров модель рассчитывает отношение **суммарных**, а не **несвязанных**
концентраций, что завышает прогнозируемую фармакологическую активность для
высокосвязывающихся соединений.

---

#### Ограничение 3: Единственный транспортёр — P-gp
**Серьёзность: Средняя**

P-gp (ABCB1) — важнейший, но не единственный эффлюксный транспортёр ГЭБ.
Для многих современных CNS-препаратов ключевую роль играют:

| Транспортёр | Гены | Значимость |
|---|---|---|
| BCRP (Breast Cancer Resistance Protein) | ABCG2 | Равноценен P-gp для многих таргетных препаратов |
| MRP4 | ABCC4 | Важен для нуклеотидных аналогов |
| LAT1 | SLC7A5 | Основной **инфлюксный** транспортёр аминокислот и их аналогов |
| OAT3 | SLC22A8 | Органические анионы |

Отсутствие BCRP критично: для ряда ингибиторов тирозинкиназ (осимертиниб, иматиниб)
двойная блокада P-gp+BCRP даёт принципиально иной фармакокинетический профиль.

---

#### Ограничение 4: Нет элиминации из кровяного компартмента
**Серьёзность: Средняя**

Модель предполагает, что вещество в крови **не выводится** (нет клиренса, нет метаболизма).
В реальности системная элиминация (печёночный + почечный клиренс) является основным
фактором, определяющим экспозицию. Без члена элиминации:

- Модель корректна только для изолированного компартмента (Transwell/PAMPA in vitro)
- При попытке описать in vivo фармакокинетику C_blood будет занижена лишь диффузией в мозг,
  а не реальным суммарным клиренсом

**Вывод:** текущая модель лучше описывает in vitro ситуацию, но нуждается в члене
клиренса для трансляции к in vivo.

---

#### Ограничение 5: Минимальный реестр веществ (4 соединения)
**Серьёзность: Средняя**

Текущий реестр: Диазепам, Верапамил, Декстран, Кофеин — 4 вещества-маркёра.
Для практического применения необходима репрезентативная база 50+ верифицированных
соединений, охватывающих:
- Различные терапевтические классы (антидепрессанты, антипсихотики, противоопухолевые ЦНС)
- Широкий диапазон logP (от -2 до +6)
- Оба полюса P-gp субстратности

---

#### Ограничение 6: Отсутствие модульных тестов
**Серьёзность: Средняя**

В репозитории нет каталога `tests/`. Это создаёт риск регрессий при добавлении
новых транспортёров и QSAR-блока. Ключевые тесты, необходимые немедленно:

```python
# Примеры критических тест-кейсов:
# 1. Закон сохранения вещества
assert abs(total_mass_t0 - total_mass_tf) < 1e-8

# 2. Граничный случай: vmax=0 (нет P-gp) → Kp,uu = 1.0 (равновесие)
assert abs(kp_uu - 1.0) < 0.01

# 3. Граничный случай: k_pass=0 (нет диффузии) → C_brain → 0
assert c_brain_max < 1e-6
```

#### Ограничение 7: Нет экспорта данных
**Серьёзность: Низкая (блокирует практическое применение)**

Результаты недоступны вне браузера: нет сохранения в CSV/Excel, нет экспорта графиков

## 3. Сравнение с аналогами

### 3.1 Российские разработки

| Параметр | PASS Online | In Silico BBB |

В России отсутствует открытая механистическая платформа для кинетического моделирования

открытая русскоязычная вычислительная платформа для механистического моделирования ГЭБ.

## 4. Анализ сильных и слабых сторон (SWOT)

### 4.1 Сильные стороны (Strengths)

| # | Сила | Значимость |
|---|---|---|
| S1 | **Механистическая кинетика** вместо QSAR — даёт C(t), Cmax, AUC, Kp,uu | Высокая |
| S2 | **Открытый код** (Python, GitHub) — воспроизводимость, модифицируемость | Высокая |
| S3 | **Веб-интерфейс** без установки — доступен любому исследователю | Высокая |
| S4 | **Нулевой порог входа**: достаточно ввести три параметра | Высокая |
| S5 | **Закон сохранения вещества** явно выполнен в ОДУ | Средняя |
| S6 | **Качественный код**: docstring, типизация, модульность | Средняя |

**Фаза: 3-компартментная модель + Валидация**

**Трёхкомпартментная модель для in vitro данных:**
```
Transwell-система:
┌──────────────────────────────────────────────────┐
│  Донорная лунка (апикальная сторона)             │
│  C_donor(t)                                      │
├──────────────────────────────────────────────────┤
│  Клетки эндотелия ГЭБ (BBB-монослой)            │
│  C_cell(t) — явный клеточный компартмент        │
├──────────────────────────────────────────────────┤
│  Акцепторная лунка (базолатеральная сторона)     │
│  C_acceptor(t)                                   │
└──────────────────────────────────────────────────┘
```

**Выходные метрики для in vitro:**
- Papp (A→B) — кажущаяся проницаемость апикаль→базолатераль [×10⁻⁶ см/с]
- Papp (B→A) — обратный транспорт
- Efflux Ratio = Papp(B→A) / Papp(A→B)

## 6. Рекомендации по развитию

### Неотложные меры (следующие 4–6 недель)

```
1. СОЗДАТЬ tests/ с pytest
   → Минимальный набор: conservation law, Kp,uu при vmax=0, edge cases
   → Команда: pip install pytest pytest-cov && pytest --cov=bbb

2. ДОБАВИТЬ экспорт CSV
   → Streamlit: st.download_button("Скачать CSV", csv_data, "results.csv")
   → 10–15 строк кода, немедленная практическая ценность

3. ЗАДОКУМЕНТИРОВАТЬ формулу k_pass ↔ Papp
   → Добавить в README и в UI (tooltip/expander)
   → Предотвращает ошибки интерпретации пользователями
```

### Стратегические рекомендации

**R1. Валидация в первую очередь, новые возможности во вторую**
Без публичной валидации на экспериментальных данных платформа остаётся
инструментом-концептом. Начать с 10–15 хорошо охарактеризованных соединений
из открытой литературы (Fridén, 2009; Hammarlund-Udenaes, 2010).

**R2. Партнёрство с лабораторией, имеющей Transwell/PAMPA установку**
Экспериментальные данные от партнёра позволят провести первичную валидацию
без необходимости самостоятельно ставить эксперименты.

**R3. Приоритизировать BCRP над MRP4**
Для современных таргетных противоопухолевых препаратов (ингибиторы тирозинкиназ,
KRAS-ингибиторы) BCRP является первичным ограничивающим транспортёром.
Добавление BCRP существенно расширит применимость.

**R4. QSAR-блок: начать с классической регрессии, не с DL**
RandomForest / XGBoost на RDKit-дескрипторах даст разумное качество при
малом обучающем наборе (<1000 соединений) и будет более интерпретируемым.
Deep learning требует 10k+ данных.

**R5. Не пренебрегать UX для экспорта**
Исследователи часто делают выбор инструмента по возможности «скопировать данные
в Excel». Экспорт CSV — ключевая функция для принятия в лабораторной практике.
